# Watchbird WAF

Watchbird 是一款专为 AWD（Attack With Defense）场景设计的强大 PHP Web 应用防火墙。

## 功能特性

### 易于部署

- 单文件架构，无需加载外部 JS/CSS 资源
- 可随时开启/关闭各项防御功能
- 提供 Web 控制台进行配置管理

### 基础防御

| 防御类型 | 说明 |
|---------|------|
| SQL 注入 | 检测并拦截数据库注入攻击 |
| 文件上传 | 限制危险文件类型上传 |
| 文件包含 | 防止本地/远程文件包含漏洞利用 |
| Flag 关键字 | 检测 flag 相关请求，防止 flag 泄露 |
| PHP 反序列化 | 拦截 unserialize 相关攻击 |
| 命令执行 | 检测 RCE 攻击尝试 |
| DDoS 防护 | 分布式拒绝服务攻击防护 |
| 请求关键字 | 检测请求头、GET/POST 参数中的危险关键字 |
| 特殊字符 | 拦截危险特殊字符 |

### 深度防御

| 防御类型 | 说明 |
|---------|------|
| 响应检测/反向代理 | 将流量发送至本地服务器自检，可配置代理服务器实现反代 |
| 响应 Flag 检测 | 检测响应中的 flag 并返回虚假 flag |
| LD_PRELOAD 保护 | 基于 LD_PRELOAD 的指令执行保护 |
| open_basedir 保护 | 基于 open_basedir 的 PHP 文件操作保护 |

### Web 控制台

- 功能开关及配置管理
- 实时日志查看
- 日志流量重放（可广播至指定网段）
- Flag 自动提取与提交
- RCE/文件上传/深度检测 防御通知

## 安装部署

### 方式一：手动部署

1. 将 `waf.so` 和 `watchbird.php` 文件放置到 Web 目录（如 `/var/www/html`）

2. 确保 `www-data` 用户对目标目录可写

3. 执行安装命令：
```bash
php watchbird.php --install /var/www/html
```

4. 安装器将输出已启用 WAF 的文件路径

5. 访问任意启用了 WAF 的文件，添加参数 `?watchbird=ui` 打开控制台

6. 创建初始密码完成配置

### 方式二：通过 AWD 防御运维工作台部署

1. 在工作台「服务器管理」中选择目标服务器

2. 点击「一键部署 WAF」

3. 选择 Watchbird WAF

4. 配置部署参数（密码、密钥）

5. 等待部署完成

### 卸载

在相同位置执行卸载命令：
```bash
php watchbird.php --uninstall /var/www/html
```

如多次安装，需多次运行卸载直到无输出。

## 控制台使用

访问已启用 WAF 的页面，添加参数：

```
http://your-site/index.php?watchbird=ui
```

首次访问需创建管理密码。

### 功能开关

在控制台中可随时开启/关闭各项防御：

- SQL 注入防护
- 文件上传防护
- 文件包含防护
- Flag 关键字检测
- 命令执行防护
- 反序列化防护
- DDoS 防护
- 深度检测

### 日志查看

- 实时查看攻击日志
- 查看被拦截的请求详情
- 流量重放功能

### 流量重放

- 将捕获的攻击流量广播至指定网段
- 支持自动提取 flag 并提交
- 可配置目标网段和提交接口

## 文件说明

| 文件 | 说明 |
|------|------|
| `watchbird.php` | WAF 主程序（单文件，包含所有功能） |
| `waf.so` | LD_PRELOAD 保护模块（C 语言编译） |

## 构建源码

如需自定义修改，可从源码构建：

```bash
# 克隆仓库
git clone https://github.com/leohearts/awd-watchbird.git

# 打包 PHP 源码为单文件
python3 pack.py

# 编译 C 模块
gcc waf.c -shared -o waf.so
```

## 安全建议

1. **及时更新密码**：首次部署后立即设置强密码
2. **定期检查日志**：监控异常攻击行为
3. **合理配置规则**：根据实际业务调整检测规则
4. **配合其他防护**：结合系统级防护（如 iptables）使用
